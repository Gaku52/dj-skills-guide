# フロアリーディング

観客の反応を読み、リアルタイムで選曲を調整するライブスキルを習得します。

## この章で学ぶこと

- フロアリーディングとは
- 観客の反応の読み方
- うまくいかない時の対処
- リアルタイムでの選曲変更
- 予備プレイリストの準備
- 失敗からの回復方法
- プロの観察術

## なぜフロアリーディングが重要か

**事前準備 vs ライブ対応:**
```
完璧なセットリスト準備:
- 流れは完璧
- でも観客の反応は？
- 計画通りにいかない時は？

フロアリーディング:
- 観客の反応を見る
- リアルタイムで調整
- 柔軟に対応
- 常に最適な選曲

プロの必須スキル
```

---

## 1. フロアリーディングとは

### 定義

```
フロアリーディング = 観客の反応を読み取るスキル

観察対象:
- ダンスしている人数
- ダンスの激しさ
- 表情
- 手の上げ方
- フロアの雰囲気

目的: 選曲を最適化
```

### DJの視点

**ブースから見える景色:**
```
前方:
- 最も熱心なダンサー
- エネルギー高い
- DJ を見ている

中央:
- メインフロア
- 大多数の観客
- ここを読む（重要）

後方:
- バー付近
- 休憩中の人
- あまり踊っていない

中央の観客が最も重要
```

---

## 2. 観客の反応の読み方

### サイン1: ポジティブな反応

**手が上がる:**
```
意味:
- 盛り上がっている
- 曲が気に入っている
- エネルギー高い

対応:
✓ このエネルギーレベルを維持
✓ 似た曲を続ける
✓ BPMをわずかに上げてもOK
```

**ジャンプ:**
```
意味:
- ピーク
- 最高潮
- 完璧なタイミング

対応:
✓ この曲を長めに
✓ ドロップリピート検討
✓ 次もハイエネルギー曲
```

**笑顔:**
```
意味:
- 楽しんでいる
- 雰囲気が良い
- リラックスしている

対応:
✓ 現状維持
✓ 急な変化は避ける
✓ 自然な流れを継続
```

**近づいてくる:**
```
意味:
- 曲名を知りたい
- DJに興味がある
- セットが気に入っている

対応:
✓ 曲名を教える
✓ コミュニケーション
✓ 次の曲も同じ方向性
```

### サイン2: ネガティブな反応

**バーに向かう:**
```
意味:
- 休憩したい
- 疲れた
- または曲が合わない

対応:
? 人数を確認（多数ならエネルギー下げる）
? 少数なら気にしない
```

**腕を組む:**
```
意味:
- 曲が合わない
- 期待と違う
- ジャンルが違う

対応:
✓ 次の曲を変更
✓ より親しみやすい曲
✓ 定番曲を入れる
```

**携帯を見る:**
```
意味:
- 退屈
- 興味がない
- 曲が合わない

対応:
✓ エネルギーを変える
✓ 有名な曲を入れる
✓ BPMを変える
```

**フロアが空く:**
```
意味:
- 明らかに失敗
- 曲の選択ミス
- エネルギーレベルのミス

対応:
✓ すぐに次の曲へ（8-16 bar）
✓ 定番曲で回復
✓ エネルギーレベルを調整
```

## 3. リアルタイムでの選曲変更

### Scenario 1: エネルギーが高すぎる

**サイン:**
```
- 多くの人がバーに向かう
- 疲れた表情
- ダンスが緩やか
```

**対応:**
```
即座に:
1. 次の曲をよりメロウに変更
2. BPMを2-3下げる
3. ビルドアップの少ない曲
4. エネルギーレベル -2

準備していたピーク曲は後回し
```

### Scenario 2: エネルギーが低すぎる

**サイン:**
```
- フロアがまばら
- あまり踊っていない
- 退屈そう
```

**対応:**
```
即座に:
1. エネルギーを上げる
2. 有名な曲を投入
3. ビルドアップのある曲
4. BPM +2-3

ウォームアップを早めに切り上げる
```

### Scenario 3: ジャンルが合わない

**サイン:**
```
- 反応が薄い
- 期待と違う表情
- フロアから離れる
```

**対応:**
```
柔軟に:
1. ジャンルを変える
2. よりメインストリームに
3. 定番曲を入れる
4. 観客層を再評価

準備していたアンダーグラウンド曲は諦める
```

## 4. 予備プレイリストの準備

### 3つの予備プレイリスト

**Plan A: メイン（準備したもの）**
```
Tech House、アンダーグラウンド
60曲

計画通りの選曲
```

**Plan B: よりメインストリーム**
```
有名な曲、定番
30曲

観客の反応が薄い場合に切り替え
```

**Plan C: 緊急用**
```
絶対に盛り上がる曲
15曲

完全に失敗した時の最終手段
例:
- Fisher - "Losing It"
- Chris Lake - "Turn Off The Lights"
- Calvin Harris - "Summer"（House remix）
```

## 5. 失敗からの回復方法

### Step 1: 失敗を認識

**サイン:**
```
- フロアが明らかに空いた
- 曲が終わるのを待っている雰囲気
- アイコンタクトがない

→ 失敗を認める（重要）
```

### Step 2: すぐに次へ

**対処:**
```
1. 失敗した曲を早めに終わらせる（16-32 bar）
2. Hot Cueで即座に次の曲へ
3. Volume Cutで切り替え
4. 定番曲で回復

時間: 30秒以内に対応
```

### Step 3: 定番曲で安定化

**回復曲の選び方:**
```
条件:
✓ 絶対に盛り上がる曲
✓ 多くの人が知っている
✓ ダンサブル
✓ ポジティブな雰囲気

例:
- Green Velvet - "La La Land"
- Fisher - "Losing It"
- その他の定番

最低3曲は必ず準備
```

### Step 4: 原因分析

**後で考える:**
```
なぜ失敗したか:
- エネルギーレベルのミス？
- ジャンルのミス？
- タイミングのミス？
- 観客層の読み違い？

次回に活かす
```

## 6. 観客タイプ別の対応

### Type 1: パーティーピーポー

**特徴:**
```
- 若い（20-25歳）
- メインストリーム好き
- 盛り上がり重視
- 有名な曲に反応
```

**選曲:**
```
✓ ポピュラーな曲多め
✓ ドロップ多用
✓ ビルドアップ重視
✓ エネルギー高め
```

### Type 2: コアな音楽ファン

**特徴:**
```
- 音楽知識豊富
- アンダーグラウンド好き
- グルーヴ重視
- 新しい曲を求める
```

**選曲:**
```
✓ アンダーグラウンド多め
✓ 掘り出し物
✓ グルーヴィー
✓ 質重視
```

### Type 3: ミックス（最も一般的）

**特徴:**
```
- 年齢層バラバラ
- 音楽嗜好多様
- バランス重要
```

**選曲:**
```
✓ メインストリーム: 40%
✓ アンダーグラウンド: 40%
✓ 定番: 20%

バランスが鍵
```

## 7. プロの観察術

### 視線の動き

**常に観察:**
```
5秒ごと:
- フロア全体をスキャン
- 中央エリア重視
- 前方と後方もチェック
- バーエリアも確認

ミックス中も観察を止めない
```

### 判断のタイミング

**いつ判断するか:**
```
曲の最初の32 bar:
→ 反応を見る
→ 次の曲を決める

ダメなら:
→ 早めに次へ（64 bar）

良ければ:
→ 予定通り続ける
```

## 8. よくある失敗と対処法

### 失敗1: 自分の好みを押し付ける

**問題:**
```
観客: Deep House希望
DJ: Techno を流す
→ フロアが空く
```

**対処:**
```
✓ 観客のために選ぶ
✓ 柔軟に対応
✓ 自分の好み: 60-70%程度
```

### 失敗2: 反応を無視する

**問題:**
```
明らかに失敗
→ 計画通り続ける
→ さらに悪化
```

**対処:**
```
✓ 反応を見る
✓ すぐに調整
✓ 計画にこだわらない
```

### 失敗3: パニックになる

**問題:**
```
失敗
→ パニック
→ さらに失敗
```

**対処:**
```
✓ 落ち着く
✓ 定番曲で回復
✓ 笑顔を保つ
✓ 観客は忘れる
```

## まとめ

- **定義**: 観客の反応を読み、リアルタイムで選曲調整
- **ポジティブサイン**: 手が上がる、ジャンプ、笑顔
- **ネガティブサイン**: バーに向かう、携帯、フロアが空く
- **リアルタイム調整**: エネルギーレベル、ジャンル、BPM
- **予備プレイリスト**: Plan A/B/C 3種類準備
- **失敗からの回復**: 認識→即次へ→定番曲→分析
- **観察**: 5秒ごと、中央エリア重視
- **柔軟性**: 計画にこだわらない、観客優先
