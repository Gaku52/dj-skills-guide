## なぜ録音が重要なのか

**プロDJの鉄則:**

```
録音しないDJ = 成長しないDJ

理由:

1. 客観的な評価
   プレイ中: 主観的、集中している
   録音: 客観的、冷静に聞ける

2. 問題点の発見
   気づかないミス、癖、弱点が明確に

3. 成長の証拠
   過去の録音と比較 → 上達を実感

4. ポートフォリオ
   良い録音 → デモ音源
           → ギグ獲得

5. フィードバック収集
   他者に聞いてもらえる
```

### トップDJも必ずやっている

```
Carl Cox:
「毎回のセットを録音して聞き直す。
 常に改善点を探している」

Richie Hawtin:
「録音は自分の先生。
 最も正直なフィードバックをくれる」

Nina Kraviz:
「自分のセットを聞くのは恥ずかしいけど、
 成長には必要不可欠」
```

## Rekordbox レコーディング設定

### 基本設定

**Preferences の設定:**

```
Rekordbox メニュー:
[rekordbox] > [Preferences] > [Controller] > [Rec]

設定項目:

1. 保存先フォルダ:
   [Browse] をクリック
   → 例: /Users/YourName/DJ/Recordings/

   推奨構造:
   /DJ/
   ├─ Recordings/
   │  ├─ Practice/      ← 練習用
   │  ├─ Sets/          ← セット録音
   │  ├─ Live/          ← ライブ録音
   │  └─ Archive/       ← アーカイブ

2. ファイル名:
   自動命名: YYYYMMDD_HHMMSS
   例: 20241206_193045.wav

   または手動:
   例: 2024-12-06_House_Practice.wav

3. フォーマット:
   □ WAV (推奨)
     - 高音質
     - 無圧縮
     - 編集しやすい
     - ファイルサイズ大

   □ AAC/MP3
     - 圧縮
     - ファイルサイズ小
     - 配信用

   推奨:
   - 練習: WAV
   - 配信アップロード用: MP3 320kbps

4. ビットレート:
   WAV: 44.1kHz / 16-bit（CD品質）
   MP3: 320kbps（高品質）

5. ショートカットキー:
   [REC] ボタン: デフォルト
   カスタム: R キー（推奨）
```

### DDJ-FLX4での録音方法

**ハードウェア操作:**

```
準備:

1. Rekordbox 起動
2. DDJ-FLX4 接続確認
3. 音量レベル確認
   [MASTER LEVEL] メーター: -6dB～0dB
   （赤ランプ点灯させない）

録音開始:

方法1: ソフトウェアボタン
├─ Rekordbox 画面上部
├─ [REC] ボタンクリック
└─ 赤く点灯 → 録音中

方法2: ショートカットキー
├─ キーボード [R] または設定したキー
└─ すぐに録音開始

録音中:
├─ [REC] ボタンが赤く点滅
├─ タイマー表示: 00:15:32
└─ 通常通りプレイ

録音停止:

1. [REC] ボタン再度クリック
   または [R] キー再度押下

2. 自動保存:
   設定したフォルダに保存

3. 確認:
   [Browse] > [Recording] タブ
   → 録音ファイル一覧
```

### 録音時の注意点

**音質を最大化:**

```
□ ゲインステージング
  各曲の TRIM:
  - レベルメーター: 中央付近
  - クリッピング（赤）絶対に避ける
  - 余裕を持たせる（-6dB～-3dB）

□ マスター音量
  [MASTER LEVEL] ノブ:
  - 12時位置（0dB）推奨
  - 赤ランプ点灯 = クリッピング = NG

□ ヘッドルーム
  - 最大音量でも -3dB 余裕
  - 後で音量調整可能
  - 歪みは修正不可

□ 環境ノイズ
  - 静かな環境
  - マイク拾わないように
  - エアコン音注意
```

## 効果的なレビュー方法

### レビューの3段階

**Stage 1: 即座レビュー（録音直後5-10分）**

```
目的: 新鮮な記憶で問題点を記録

手順:

1. 録音終了直後（5分以内）
2. メモを取る:
   □ 全体的な感想
   □ 明らかなミス箇所
   □ 良かった瞬間
   □ 次回の改善点

3. テキストファイル作成:
   ファイル名: 録音と同じ
   例: 2024-12-06_Practice_NOTES.txt

   内容例:
   ```
   2024-12-06 Practice Session

   Total: 60min
   Tracks: 15

   良かった点:
   - ビートマッチング安定
   - 5曲目→6曲目のトランジション最高

   問題点:
   - 15分頃、ビートずれ
   - 30分頃、EQバランス悪い
   - 選曲の流れが単調

   次回:
   - Techno曲を追加
   - エフェクト練習
   ```
```

**Stage 2: 詳細レビュー（翌日 30-60分）**

```
目的: 客観的に全体を分析

準備:
□ ヘッドフォン（良い音質で）
□ メモ帳 / PC
□ タイマー
□ 集中できる環境

手順:

1. 最初から最後まで通して聞く（1回目）
   - メモを取りながら
   - タイムスタンプ記録

   タイムスタンプメモ例:
   ```
   00:00 - イントロ良好
   05:30 - ビートずれ（要注意）
   08:15 - 良いトランジション
   12:45 - EQ MID 上げすぎ
   18:00 - エネルギー低下
   25:30 - 最高のミックス！
   32:00 - 選曲ミス?流れ悪い
   45:00 - クライマックス成功
   55:00 - エンディング良好
   ```

2. 問題箇所を再度聞く（2回目）
   - 05:30 のビートずれを詳細分析
   - 何が原因？
   - どう修正すべき？

3. 評価シート記入:
   ビートマッチング: __/10
   ミキシング: __/10
   選曲: __/10
   エナジーカーブ: __/10
   全体: __/10

4. アクションプラン作成:
   次回の重点課題リスト
```

**Stage 3: 週次レビュー（日曜 30分）**

```
目的: 成長を確認、長期計画

手順:

1. 今週の全録音を確認
   - 何本録音した？
   - 総録音時間？
   - ベスト録音は？

2. 成長ポイント発見
   - 月曜 vs 金曜で比較
   - 改善した点は？
   - まだ課題は？

3. データ記録
   Googleスプレッドシートなどに:
   ```
   Week 1:
   - 録音本数: 5
   - 総時間: 4.5h
   - ビートマッチング平均: 6/10
   - ベストセット: 金曜日
   ```

4. 来週の計画
   - 重点課題
   - 新しい挑戦
```

## タイムスタンプメモの取り方

### 効率的なメモ技術

**記号システム:**

```
✓ = 良い / 成功
✗ = 悪い / 失敗
★ = 素晴らしい / 完璧
! = 要注意 / 重要
? = 疑問 / 確認必要

使用例:

00:00 - イントロ ✓
05:30 - ビートずれ ✗!
08:15 - トランジション ★
12:45 - EQ問題 ✗
18:00 - エナジー低下 ?
25:30 - 最高のミックス ★★★
32:00 - 選曲ミス ✗!
45:00 - クライマックス ✓
55:00 - エンディング ✓
```

### デジタルツール活用

**Rekordbox マーカー機能:**

```
録音ファイルにマーカー追加:

1. [Browse] > [Recording] で録音選択
2. ダブルクリックで波形表示
3. 問題箇所で右クリック
   → [Add Memory Cue]
4. メモを入力
   例: "ビートずれ - BPM調整不足"

5. マーカー一覧:
   - 色分け可能
   - メモ付き
   - 後で簡単に見つかる

色分け例:
赤: 問題箇所
緑: 良いミックス
黄: 改善余地あり
青: メモ
```

**スマホアプリ:**

```
ボイスレコーダーアプリ:

練習中に気づいたこと:
→ スマホで即座に音声メモ

「15分頃、ビートがずれた。
 次の曲のBPMが予想より速かった」

後で文字起こし:
- Otter.ai
- Google Recorder
- iPhone ボイスメモ（自動文字起こし）
```


## アップロードと公開

### SoundCloud へのアップロード

**準備:**

```
1. WAV → MP3 変換（必要な場合）
   - Audacity（無料）
   - GarageBand（Mac）
   - オンライン変換ツール

   設定:
   - フォーマット: MP3
   - ビットレート: 320kbps
   - サンプルレート: 44.1kHz

2. カバーアート作成
   - サイズ: 800×800px 以上
   - Canva（無料）
   - Photoshop

   内容:
   - DJ名
   - セットタイトル
   - 日付
   - ジャンル

3. トラックリスト作成
   ```
   Tracklist:

   1. Artist1 - Track1
   2. Artist2 - Track2
   3. Artist3 - Track3
   ...
   ```
```

**アップロード手順:**

```
SoundCloud:

1. ログイン
   https://soundcloud.com/

2. [Upload] クリック

3. ファイル選択
   MP3 をドラッグ&ドロップ

4. 情報入力:

   Title:
   「[Progressive House] 60min Set - December 2024」

   Genre:
   House / Techno / Hip Hop など

   Tags:
   #dj #house #mix #progressivehouse #djset
   （最大30個）

   Description:
   ```
   Progressive House 60min Set
   Recorded: 2024-12-06

   Tracklist:
   1. Artist1 - Track1
   2. Artist2 - Track2
   ...

   Follow me:
   Instagram: @yourdjname
   Mixcloud: /yourdjname

   #dj #house #progressivehouse
   ```

5. 設定:
   Privacy: Public / Private

   初心者: Private（練習用）
   中級者以上: Public（公開）

   Download: Enable（聴いてもらいやすい）

6. [Save] → [Publish]
```

### Mixcloud へのアップロード

**Mixcloud の利点:**

```
SoundCloud vs Mixcloud:

SoundCloud:
- DJ以外の音楽全般
- 制限: 著作権厳しい
- Free: 3時間まで

Mixcloud:
- DJ Mix 特化
- 著作権: DMCA対応済み
- 長時間OK
- ライセンス料込み

→ DJ Mix は Mixcloud 推奨
```

**アップロード手順:**

```
1. https://www.mixcloud.com/

2. [Upload] クリック

3. ファイルアップロード
   MP3 / M4A

4. 情報入力:

   Title:
   「Progressive House Journey - December 2024」

   Description:
   同様にトラックリスト記載

   Tags:
   House, Progressive House, DJ Mix

   Tracklist（重要！）:
   - 自動認識
   - 手動で修正
   - 各曲のアーティスト名とタイトル

   ※ トラックリスト必須
   　→ 著作権対応のため

5. カバーアート
   800×800px 以上

6. [Publish]
```

## 録音の保存とバックアップ

### バックアップ戦略

**3-2-1ルール:**

```
3: 3つのコピー
2: 2つの異なるメディア
1: 1つはオフサイト

実践例:

コピー1: Mac / PC（オリジナル）
コピー2: 外付けHDD
コピー3: クラウド（Google Drive / Dropbox）

定期的にバックアップ:
- 毎週: 外付けHDD
- 毎月: クラウド

重要な録音は:
即座にバックアップ
```

### ストレージ計算

**必要容量:**

```
WAV録音:

60分 = 約600MB
90分 = 約900MB
120分 = 約1.2GB

週2回録音（各60分）:
週: 1.2GB
月: 5GB
年: 60GB

外付けHDD:
1TB で約1,600本（60分）の録音

推奨:
- 外付けHDD: 1-2TB
- クラウド: 100GB-1TBプラン
```

## よくある質問

### Q1: 毎回録音すべき？

**A:** はい、できる限り全て録音

```
理由:
- 客観的な評価のため
- 成長の記録
- 思わぬ良いミックスが生まれる

最低限:
- 週1回のメインセット
- ライブ/配信は必須
```

### Q2: 録音を聞き直すのが恥ずかしい

**A:** 誰もが通る道、慣れる

```
考え方:
- 完璧なDJはいない
- プロも失敗している
- 改善のため必要

ヒント:
- 最初は良い部分だけ聞く
- 徐々に問題箇所も分析
- 成長を実感すると楽しくなる
```

### Q3: ファイルサイズが大きすぎる

**A:** MP3に変換、または古い録音削除

```
対策:

1. WAV → MP3 変換
   60分WAV: 600MB
   60分MP3: 140MB
   約4分の1に

2. 保存期間設定
   - 練習録音: 3ヶ月保存
   - 良いセット: 永久保存
   - 定期的に整理

3. クラウドストレージ
   - Google Drive
   - Dropbox
   - OneDrive
```
