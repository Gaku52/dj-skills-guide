## デジタルオーディオとは

**アナログからデジタルへ:**

### アナログ音声

```
現実世界の音:

空気の振動:
連続的な波

アナログ録音:
カセットテープ
レコード

特徴:
滑らかな曲線
劣化する

デジタル化の理由:
PC で扱える
劣化しない
編集自由
```

### サンプリング

```
アナログ → デジタル変換:

1秒間に何回測定するか:
= サンプルレート

各測定値の精度:
= ビット深度

例:

CD音質:
44,100回/秒 測定
各測定値 16bit

イメージ:

アナログ:  ~~~~~~~~~~~~
           滑らかな曲線

デジタル:  ■■■■■■■■■■
           階段状（実際は非常に細かい）

人間の耳:
44.1kHz あれば滑らかに聞こえる
```

## サンプルレート

**1秒間の測定回数:**

### 主要なサンプルレート

```
22,050 Hz (22.05 kHz):
低音質
電話レベル
使わない

44,100 Hz (44.1 kHz): ← 標準
CD音質
ほぼ全ての音楽配信
推奨

48,000 Hz (48 kHz):
ビデオ標準
映像制作
やや高音質

88,200 Hz (88.2 kHz):
ハイレゾ
2倍精度
必要性低い

96,000 Hz (96 kHz):
プロ用
2倍精度
CPU負荷高い

192,000 Hz (192 kHz):
オーバースペック
4倍精度
実用性低い

推奨:

音楽制作: 44.1 kHz
映像制作: 48 kHz
それ以外: 不要
```

### ナイキスト周波数

```
理論:

サンプルレート / 2 = 最大周波数

44.1 kHz の場合:
44,100 / 2 = 22,050 Hz

意味:
最大 22,050 Hz まで記録可能

人間の可聴域:
20 Hz - 20,000 Hz

結論:
44.1 kHz で十分
人間の可聴域を完全カバー

高いサンプルレート不要:
96 kHz → 最大 48 kHz
→ 人間には聴こえない
→ 無駄
```

## ビット深度

**各サンプルの精度:**

### ビット深度の種類

```
8 bit:
256段階
レトロゲーム音質
使わない

16 bit: ← CD標準
65,536段階
十分な音質
一般的

24 bit: ← プロ標準
16,777,216段階
非常に高精度
録音時推奨

32 bit (float):
無限大のダイナミックレンジ
処理用
最終書き出しには不要

推奨:

録音時: 24 bit
→ 余裕を持たせる

書き出し時: 16 bit
→ ファイルサイズ削減

制作中: 32 bit (内部処理)
→ Abletonが自動

なぜ24bitで録音:

ヘッドルーム:
音量に余裕
クリッピング防止

後処理:
編集の余地

ノイズフロア:
低いノイズ
```

### ダイナミックレンジ

```
定義:
最小音 〜 最大音 の範囲

計算:
ビット深度 × 6 dB

16 bit:
16 × 6 = 96 dB

24 bit:
24 × 6 = 144 dB

比較:

ささやき声: 30 dB
通常の会話: 60 dB
ライブ会場: 110 dB
ジェット機: 130 dB

結論:
16 bit (96 dB) で十分
24 bit (144 dB) は余裕あり
```

## ファイル形式

**形式別の特徴:**

### 非圧縮形式

```
WAV (Waveform Audio File):

特徴:
無圧縮
Windows標準
最も一般的

音質:
最高（ロスレス）

ファイルサイズ:
大きい
44.1kHz/16bit → 約10MB/分

用途:
制作用
マスター保存

AIFF (Audio Interchange File Format):

特徴:
無圧縮
Mac標準
WAVと同等

音質:
最高（ロスレス）

ファイルサイズ:
WAVと同じ

用途:
Mac環境
制作用

FLAC (Free Lossless Audio Codec):

特徴:
可逆圧縮
音質劣化なし

ファイルサイズ:
WAVの50-70%

用途:
アーカイブ
配信（一部）
```

### 圧縮形式

```
MP3:

特徴:
非可逆圧縮
古い規格
互換性最高

音質:
320kbps: 良い
192kbps: 普通
128kbps: 悪い

ファイルサイズ:
320kbps → 約2.5MB/分

用途:
Demo配信
DJ用（一部）

AAC:

特徴:
MP3の後継
Apple標準
効率良い

音質:
256kbps AAC = 320kbps MP3

ファイルサイズ:
小さい

用途:
iTunes
Apple Music

OGG Vorbis:

特徴:
オープンソース
高効率

音質:
良い

用途:
Spotify
ゲーム

推奨:

制作中: WAV 44.1kHz/24bit
書き出し: WAV 44.1kHz/16bit
配信用: MP3 320kbps
```

## Audio Clipの構造

**Ableton での表示:**

### Clip View

```
Audio Clip:

┌──────────────────────────────┐
│ Clip Name: Kick_001.wav      │
├──────────────────────────────┤
│ Waveform:                    │
│ ~~~~~~~~~/\~~~~~~~~~/\~~~~~~ │ ← 波形表示
│                              │
│ Loop: [====================] │ ← ループ範囲
│ Start: 1.1.1    End: 1.5.1   │
│                              │
│ Transpose: 0 st              │
│ Detune: 0 ct                 │
│                              │
│ Warp: ☑ On                   │
│ BPM: 128.00                  │
└──────────────────────────────┘

各要素:

Waveform:
音の視覚表現
縦軸: 振幅（音量）
横軸: 時間

Loop:
繰り返し範囲
Start/End で指定

Transpose:
半音単位で移調
±12 = 1オクターブ

Detune:
セント単位で微調整
100 cent = 1半音

Warp:
BPM同期
詳細は warp-modes.md
```

### Sample設定

```
Clip View > Sample:

Hi-Q: ☑
高音質モード
CPU負荷高い
推奨: On

Preserve:
ピッチ保存
Transpose時の音質

Ram:
メモリに読み込み
大量サンプル時は Off

Reverse:
逆再生
エフェクト的

Fade:
Fade In/Out
クリック音防止
```

## 実践: Audioを読み込む

**基本操作:**

### Step 1: サンプルをインポート (5分)

```
方法1: Browser からドラッグ

1. Browser 開く (Cmd+Opt+B)

2. Places > Packs > Core Library

3. Drums > Kicks > Kick.wav

4. Session View の空スロットにドラッグ

5. クリップ作成完了

方法2: ファイル直接ドラッグ

1. Finder/Explorer でWAVファイル

2. Ableton にドラッグ

3. 自動的にトラック作成

方法3: Place > Insert Audio Clip

1. トラック選択

2. 右クリック > Insert Audio Clip

3. ファイル選択
```

### Step 2: 波形確認 (5分)

```
1. クリップをダブルクリック
   → Clip View 表示

2. 波形を観察:

大きい波:
音量が大きい

小さい波:
音量が小さい

密集:
高い音

疎:
低い音

3. 再生 (Space)
   → 聴きながら波形を見る

4. Zoom:
   + / - キー
   → 細部まで確認
```

### Step 3: ループ設定 (5分)

```
1. Loop Brace:
   黄色いバー
   ドラッグで範囲設定

2. 開始点:
   バーの左端

3. 終了点:
   バーの右端

4. Length:
   4 Bars（4小節）に設定

5. 再生:
   ループ再生される

6. 調整:
   完璧にループするまで微調整
```

## よくある質問

### Q1: 44.1kHz と 48kHz、どっちが良い？

**A:** 音楽なら44.1kHz

```
44.1 kHz:
CD音質
音楽配信標準
Spotify, Apple Music

48 kHz:
ビデオ標準
YouTube, 映画

選び方:

音楽のみ: 44.1 kHz
映像用: 48 kHz
迷ったら: 44.1 kHz

混在:

プロジェクト内で統一:
44.1 kHz のみ
または
48 kHz のみ

混ぜない:
サンプルレート変換
= 音質劣化の可能性
```

### Q2: 録音は24bitでやるべき？

**A:** Yes

```
録音時:

24 bit推奨:
ヘッドルーム大きい
後処理の余地

16 bitでも:
可能だが余裕なし

書き出し時:

16 bit:
CD音質
ファイルサイズ小
配信用

24 bit:
マスター保存用
レーベル提出用

流れ:

録音: 24 bit
制作: 32 bit (内部)
書き出し: 16 bit (配信)
          24 bit (マスター)
```

### Q3: MP3は制作に使える？

**A:** 推奨しない

```
理由:

非可逆圧縮:
音質劣化
元に戻らない

編集:
MP3 → 編集 → MP3
= 2重圧縮
= さらに劣化

推奨:

制作中: WAV のみ
書き出し後: MP3変換

やむを得ない場合:

320kbps MP3:
最低限の音質
デモ制作のみ

プロ:
MP3 使わない
全て WAV
```
