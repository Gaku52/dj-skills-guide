## フィードバック収集方法

**効率的な収集:**

### Step-by-Step

```
Step 1: 準備（v1.0完成）

完成度:
80%以上
聴ける状態

Export:
WAV → MP3
128-320 kbps

プライベートリンク:
SoundCloud Private
Google Drive

Step 2: 自分で聴く（1週間）

完成:
保存

1週間:
触らない
聴かない

1週間後:
聴き直す

メモ:
改善点記録

修正:
大きな問題のみ
v2.0作成

Step 3: プロデューサーに送る（3-5人）

メッセージ:

件名: 「Feedback Request - Track Name」

本文:
「Hi, I just finished a new track and would love your feedback.

Track: [Link]
Genre: Techno
BPM: 128
Length: 6:00

Specific questions:
1. How's the mixdown?
2. Is the arrangement engaging?
3. Any technical issues?

Thanks!」

具体的質問:
答えやすい

返信率:
30-50%

Step 4: DJに送る（2-3人）

メッセージ:

「Hi, I made a techno track and wondering if it's DJ-friendly.

Track: [Link]

Questions:
1. Would you play this in your set?
2. Is the intro/outro long enough?
3. Any issues for DJing?

Thanks!」

DJ視点の質問:
明確

Step 5: 一般リスナー（5-10人）

メッセージ:

「Hey! Made a new track, what do you think?

Track: [Link]

Questions:
1. Do you like it?
2. Does it get boring?
3. Overall impression?」

シンプル:
答えやすい

Step 6: 収集・記録

Notion・Excel:

| Source | Name  | Type      | Feedback                  | Priority |
|--------|-------|-----------|---------------------------|----------|
| Producer | John | Technical | Kick too loud           | High     |
| Producer | Mike | Technical | Bass low-mid muddy      | High     |
| DJ     | Sarah | Practical | Intro too short         | High     |
| Listener | Tom | General  | Love it!                 | -        |
| Self   | Me    | Overall   | Drop too early          | Medium   |

整理:
優先順位つける
```

## フィードバックの分析

**何を信じるか:**

### フィードバック分類

```
分類1: 技術的 vs 好み

技術的:

例:
「Kickがクリッピング」
「Bassが小さい」
「Mix濁ってる」
「Stereo Phase問題」

特徴:
客観的
測定可能
明確

対応:
必ず修正
優先度: 高

好み:

例:
「このジャンル嫌い」
「もっと明るい曲好き」
「Lead音色が好みじゃない」

特徴:
主観的
個人の好み
曖昧

対応:
参考程度
優先度: 低

分類2: 複数人 vs 1人

複数人（3人以上）:

例:
「Bassが小さい」
5人中4人指摘

→ 必ず修正
客観的に問題あり

1人のみ:

例:
「Bassが小さい」
5人中1人指摘

→ 参考程度
その人の好み可能性

ルール:

3人以上:
必ず修正

2人:
検討

1人:
無視してOK（好みの場合）

分類3: 具体的 vs 曖昧

具体的:

例:
「Bar 73のKickが歪んでる」
「2:30のBassが小さい」
「Hi-Hatが右に寄りすぎ」

特徴:
明確
修正容易

対応:
優先修正

曖昧:

例:
「なんか違う」
「イマイチ」
「微妙」

特徴:
不明確
修正困難

対応:

質問:
「具体的にどこ？」

無視:
改善できない

分類4: ソース別重み

プロデューサー:
重み 100%
技術的
最優先

DJ:
重み 80%
実践的
重要

リスナー:
重み 30%
感覚的
参考

自分:
重み 100%
正直
重要

優先順位:

1. プロデューサー技術的指摘
2. 自分の客観的問題
3. DJ実践的指摘
4. 複数人の共通指摘
5. 具体的な指摘
6. 好みの指摘（低優先）
```

```
「わかってない」

「お前の曲もダメだろ」

無視:
「好みの問題」
全て無視

感情的:
怒る
落ち込む
やる気なくす

OK反応:

感謝:
「ありがとう」
「参考になる」

質問:
「具体的にどこ？」
「どう改善すべき？」

「確かに」
「聴き直してみる」

前向き:
「次のバージョンで修正」
「成長の機会」

重要:

プロも批判される:
当たり前

批判歓迎 = 成長加速

実践:

Step 1: 全て聴く
感情的にならない

Step 2: メモする
記録

感情冷める

Step 4: 分析
優先順位

Step 5: 修正
改善
```

## イテレーションサイクル

**改善の繰り返し:**

### 3回ルール

```
Iteration 1: v1.0 → v2.0

Step 1: v1.0完成
自分で80%完成

Step 2: 自分で聴く（1週間後）
問題発見
メモ

Step 3: 他人に送る
フィードバック収集

Step 4: 分析
優先順位つける

Step 5: 修正
v2.0作成

時間:
2-3日

変更:
大きな問題のみ
技術的問題優先

Iteration 2: v2.0 → v3.0

Step 1: v2.0完成
大きな問題修正済み

Step 2: 再度フィードバック
同じ人または新しい人

Step 3: 分析
残りの問題

Step 4: 微調整
v3.0作成

時間:
1-2日

変更:
細かい調整
微調整

Iteration 3: v3.0 → Final

Step 1: v3.0完成
ほぼ完璧

Step 2: 最終確認
自分・他人

Step 3: 最終調整
本当に小さい修正

Step 4: Final
リリース

時間:
1日

変更:
最小限

3回制限:

理由:
4回以上 = 時間の無駄
改善率低い
完璧主義の罠

例外:
初めての10曲
4-5回OK
学習期間

11曲目以降:
3回厳守

合計時間:

v1.0作成: 20時間
Iteration: 4-6時間

合計: 24-26時間

許容範囲
```

## 改善の優先順位

**何から直すか:**

### 優先順位マトリックス

```
優先度1（最優先）:

技術的問題 + 複数人指摘:

例:
「Kickがクリッピング」
3人指摘

対応:
即座に修正

優先度2（高優先）:

技術的問題 + 1人指摘:

例:
「Bassが小さい」
1人指摘

対応:
自分で確認
問題あれば修正

優先度3（中優先）:

実践的問題 + DJ指摘:

例:
「Intro短すぎ」
DJ指摘

対応:
修正
DJ対応重要

優先度4（低優先）:

好み + 複数人指摘:

例:
「もっと明るい曲がいい」
3人指摘

対応:
参考程度
次の曲で考慮
この曲は変えない

優先度5（無視）:

好み + 1人指摘:

例:
「このジャンル嫌い」
1人指摘

対応:
無視

実践例:

フィードバック収集結果:

1. 「Kick clipping」（プロデューサー3人） → 優先度1
2. 「Bass too quiet」（プロデューサー1人） → 優先度2
3. 「Intro 16 bars only」（DJ 2人） → 優先度3
4. 「Too dark」（リスナー3人） → 優先度4
5. 「Don't like techno」（リスナー1人） → 優先度5

修正順序:

1番 → 2番 → 3番
4番・5番は無視

時間配分:

優先度1: 2時間
優先度2: 1時間
優先度3: 1時間

合計: 4時間
v2.0完成
```

## まとめ

### Feedback & Iteration

```
□ フィードバックソース4つ
□ プロデューサー最優先
□ 収集方法マスター
□ 分析・優先順位
□ 建設的に受け止める
□ イテレーション3回
□ 改善優先順位明確
```

### 重要原則

```
□ フィードバック = ギフト
□ 技術的問題 = 優先修正
□ 複数人指摘 = 必須修正
□ 好みの指摘 = 参考程度
□ 3回イテレーション = 最適
□ 感謝・前向き = マインド
```

### 効果

```
成長速度: 3-5倍
プロレベル到達: 2-3年
完成度: 大幅向上
客観性: 獲得
```
