# Finishing Tracks（楽曲完成技術）

楽曲を確実に完成させる技術、完成基準、完璧主義の罠を回避。完成率を80%まで引き上げます。

## この章で学ぶこと

- 「完成」の定義
- 完成基準6項目
- 完璧主義の罠
- 20時間ルール
- リリース判断
- 諦める技術
- 完成させる習慣
- プロの完成戦略

## 「完成」の定義

**完成とは何か:**

### 完成の誤解

```
NG定義:

「完璧」:
存在しない
永遠に完成しない

「満足」:
主観的
基準曖昧

「最高傑作」:
次があると信じる
現在を完成させない

OK定義:

「リリース可能」:
客観的
明確

「完成基準6項目クリア」:
具体的
判断容易

「これ以上改善点なし」:
実際的
時間制限内で

重要:

完成 = リリースできる状態
完璧 = 存在しない幻想

完成させる = スキル
完璧を目指す = 病気
```

## 完成基準6項目

**客観的な判断:**

### チェックリスト

```
1. チェックリスト全てYes

Technical:
□ クリッピングなし
□ 目標LUFS達成
□ Mono再生OK
□ 全デバイスOK（スマホ・ヘッドホン・スピーカー）

Musical:
□ 全セクション完成（Intro-Outro）
□ エネルギーカーブ理想的
□ トランジション自然
□ 飽きずに最後まで聴ける

判断:
全てYes → 次の基準へ
1つでもNo → 修正

2. Reference曲と同等

比較項目:

Volume Balance:
Reference ≒ 自分の曲

Frequency Balance:
Reference ≒ 自分の曲
（Analyzer使用）

Stereo Width:
Reference ≒ 自分の曲

Depth:
Reference ≒ 自分の曲

Energy:
Reference ≒ 自分の曲

判断:
80%以上同等 → OK
80%未満 → 修正必要

重要:
100%同じ不要
80%で十分

3. Mono再生で問題なし

テスト:

Utility:
Width 0%（Mono）

再生:
全セクション

確認:
□ 音が消えない？
□ バランス変わらない？
□ 違和感ない？

理由:
クラブ・ラジオ
Mono再生多い

Mono NG:
リリース不可

4. 一晩寝かせて聴き直してOK

方法:

Step 1: 完成と思う

Step 2: 保存・Export

Step 3: 1晩寝る
触らない

Step 4: 翌日聴く
新鮮な耳

判断:

「良い！」:
完成

「ここダメ」:
明確な問題あり
修正

「なんか違う」:
主観
無視してOK

重要:
大きな問題なし = 完成
小さな違和感 = 無視

5. 他人に聴かせてOK

方法:

プロデューサー仲間:
2-3人

聴かせる:
意見を求める

フィードバック:

3人以上が同じ指摘:
修正必須

1人だけの指摘:
参考程度

技術的指摘:
優先修正

好みの指摘:
無視してOK

判断:

致命的な問題なし:
完成

大きな問題あり:
修正

小さな問題:
次の曲で改善
この曲は完成

6. これ以上改善点が見つからない

状態:

変更するほど悪化:
完成している証拠

小さな修正の繰り返し:
完璧主義の罠

新しいアイデアが湧く:
新曲で実現
この曲は完成

判断:

1時間考えて改善点なし:
完成

すぐ改善点見つかる:
未完成

重要:
「見つからない」=「ない」
ではなく
「今の自分には見つからない」
それで完成
```

## 20時間ルール

**強制完成:**

### ルールの内容

```
ルール:

20時間制作したら:
必ず完成させる
または
諦めて次へ

例外なし

理由:

20時間以上:
改善率低い
時間対効果悪い

新しい曲:
学びが多い
成長早い

統計:
20時間で完成 = 完成率80%
40時間かける = 完成率20%

実装:

時間記録:

Phase 1: 2時間
Phase 2: 3時間
Phase 3: 2.5時間
Phase 4: 2.5時間
Phase 5: 4時間
Phase 6: 4時間
Phase 7: 2時間

合計: 20時間

判断:

20時間経過:

気に入っている:
→ Phase 7完成させる
→ リリース

気に入らない:
→ 諦める
→ Archive
→ 次の曲

20時間未満:

続ける
完成目指す

20時間超えそう:

残り1-2時間で完成できる:
→ OK

残り5時間以上必要:
→ 諦める

例外:

初めての10曲:
30-40時間OK
学習期間

11曲目以降:
20時間厳守

重要:

諦める = 失敗ではない
諦める = 賢明な判断
次の曲で活かす

20時間で完成できない曲:
根本的な問題あり
作り直した方が早い
```
